## 第二步：部署后端到 Railway

### 2.1 注册 Railway

1. 打开 https://railway.app
2. 点击 **Start for Free**
3. 选择 **Continue with GitHub**
4. 授权 Railway 访问你的 GitHub 账号

### 2.2 创建项目

1. 登录后点击 **New Project**
2. 选择 **Deploy from GitHub repo**
3. 找到并选择 `traceability-system` 仓库
4. 点击 **Add**

### 2.3 添加 PostgreSQL 数据库

1. 在项目页面点击 **New** → **Database** → **Add PostgreSQL**
2. 等待数据库创建完成（约 1 分钟）
3. Railway 会自动将数据库连接信息注入到环境变量

### 2.4 重新部署应用

1. 点击 **traceability-system** 服务
2. 在 **Deployments** 标签页点击 **Redeploy**
3. 等待部署完成（状态变为 ✅）

### 2.5 获取 API 地址

1. 部署完成后，点击 **Settings** → **Networking**
2. 找到 **Public Domain**，类似：
   ```
   https://traceability-system-production-2209.up.railway.app
   ```
3. 复制这个地址，后面要用

### 2.6 初始化示例数据

1. 在 Railway 控制台点击 **traceability-system** 服务
2. 点击 **Shell** 标签页
3. 输入命令：
   ```bash
   python init_data.py
   ```
4. 看到 "示例数据创建成功" 即可

## 第三步：部署前端到 GitHub Pages

### 3.1 修改 API 地址

**重要！** 需要把前端代码里的 API 地址改成你的实际地址。

1. 打开 GitHub 仓库
2. 找到 `index.html`（根目录下的）
3. 点击编辑按钮（铅笔图标）
4. 找到第 30 行左右：
   ```javascript
   const API_BASE = 'https://traceability-system-production-2209.up.railway.app';
   ```
5. 确认地址正确（如果不正确，改成你刚才复制的地址）
6. 点击 **Commit changes**

### 3.2 启用 GitHub Pages

1. 在 GitHub 仓库页面，点击 **Settings**
2. 左侧菜单找到 **Pages**（在 Code and automation 下面）
3. **Source** 选择：
   - Branch: `main`
   - Folder: `/(root)`
4. 点击 **Save**

### 3.3 等待部署

- 页面会显示：`Your site is ready to be published at ...`
- 等待 2-3 分钟
- 刷新页面，看到 `Your site is live at ...` 表示成功

### 3.4 访问前端页面

你的前端地址是：
```
https://你的用户名.github.io/traceability-system/
```

例如：
```
https://koolboy1978-ctrl.github.io/traceability-system/
```

## ❓ 常见问题

### Q1: Railway 部署失败怎么办？

**检查清单：**
- [ ] 仓库是否包含 `requirements.txt`
- [ ] `railway.json` 配置是否正确
- [ ] 数据库是否已添加并关联

**查看日志：**
1. 在 Railway 控制台点击 **Deployments**
2. 点击失败的部署
3. 查看 **Logs** 找错误信息

### Q2: 前端页面显示空白？

**可能原因：**
- API 地址配置错误
- 浏览器缓存问题

**解决方法：**
1. 确认 `index.html` 中的 `API_BASE` 地址正确
2. 按 `Ctrl+F5` 强制刷新页面
3. 打开浏览器开发者工具（F12）查看 Console 错误

### Q3: 查询时提示 "产品不存在"？

**原因：** 数据库中没有这个产品编码

**解决方法：**
1. 先运行 `python init_data.py` 创建示例数据
2. 或者添加你自己的产品数据

### Q4: GitHub Pages 访问很慢？

**正常现象：** GitHub Pages 在国内访问速度一般

**优化方案：**
- 使用 CDN 加速（如 Cloudflare）
- 或者把前端也部署到 Railway

### Q5: 如何修改页面样式？

编辑 `index.html` 文件：
- 样式在 `<style>` 标签内
- 可以修改颜色、字体、布局等
- 修改后提交到 GitHub，自动重新部署
