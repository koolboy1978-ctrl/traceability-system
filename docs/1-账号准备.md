## 步骤 1：上传代码到 GitHub

### 1.1 下载代码压缩包

我已将完整代码放在项目目录：
```
/Users/apple/.qclaw/workspace-agent-2a337e68/traceability-system/
```

请把这个文件夹压缩成 ZIP 文件，或者直接上传到 GitHub。

### 1.2 上传到 GitHub

有两种方式，推荐方式一：

**方式一：使用 GitHub 网页上传（简单）**

1. 打开你的 GitHub 仓库：`https://github.com/YOUR_USERNAME/traceability-system`
2. 点击 "Add file" → "Upload files"
3. 把整个 `traceability-system` 文件夹里的**所有文件**拖进去
4. 点击 "Commit changes"

**方式二：使用 Git 命令行**

如果你装了 Git，在终端执行：
```bash
cd ~/.qclaw/workspace-agent-2a337e68/traceability-system
git init
git add .
git commit -m "v1.0.0 - 农产品溯源系统初始版本"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/traceability-system.git
git push -u origin main
```

## 步骤 2：部署后端到 Railway

### 2.1 注册 Railway

1. 打开 [railway.app](https://railway.app)
2. 点击 "Sign Up" → 用 GitHub 账号登录
3. 授权 Railway 访问你的 GitHub

### 2.2 创建项目

1. 点击 "New Project" → "Deploy from GitHub repo"
2. 选择你的 `traceability-system` 仓库
3. Railway 会自动检测到 `railway.json` 配置文件

### 2.3 配置环境变量（可选）

如果需要固定 API 地址：
1. 点击项目 → "Settings" → "Networking"
2. 勾选 "Generate Domain" 自动生成免费域名

### 2.4 等待部署完成

- 状态变成 ✓ 绿色表示成功
- 底部会显示访问地址，如：`https://traceability-api.up.railway.app`

### 2.5 初始化示例数据

部署完成后，访问：
```
https://你的API地址/api/v1/init
```
（如果我添加了这个接口的话）

或者直接在 Railway 控制台运行：
1. 点击项目 → "Debug"
2. 输入命令：`python init_data.py`

## 步骤 3：部署前端到 GitHub Pages

### 3.1 启用 GitHub Pages

1. 打开 GitHub 仓库设置
2. 左侧菜单找到 "Pages"
3. Source 选择 `main` 分支，`/ (root)` 目录
4. 点击 "Save"

### 3.2 修改前端 API 地址

部署前需要修改 `frontend/index.html` 中的 API 地址：

```javascript
// 把这行
const API_BASE = window.location.hostname === 'localhost' 
    ? 'http://localhost:8000' 
    : 'https://YOUR-API-URL.up.railway.app';

// 改成你 Railway 的实际地址，例如：
const API_BASE = 'https://traceability-api.up.railway.app';
```

### 3.3 提交更新

```bash
git add .
git commit -m "v1.0.0 - 配置API地址"
git push
```

### 3.4 等待部署

几分钟后，你的网站就会上线：
```
https://YOUR_USERNAME.github.io/traceability-system/
```

## ❓ 遇到问题？

**Q: Railway 部署失败？**
- 检查 `backend/requirements.txt` 是否存在
- 检查 Python 版本（推荐 3.11）

**Q: 前端无法访问 API？**
- 确认 API 地址已正确修改
- 检查 Railway 的 CORS 设置

**Q: GitHub Pages 404？**
- 确认分支和目录选择正确
- 等待几分钟让部署完成
